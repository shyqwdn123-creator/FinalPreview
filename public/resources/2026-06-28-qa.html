Q1 13:02 知识点
虚拟内存：请求调页与页面置换的定义及实现细节
请求调页 + 页面置换

请求调页 (Demand Paging)：程序运行时，只有被访问的页面才从磁盘调入内存，按需加载，基于局部性原理。

页面置换 (Page Replacement)：物理内存满时，选择牺牲页换出，为新页腾空间。

  • 算法：FIFO（可能Belady异常）、LRU（硬件开销大）、Clock（最常用）、OPT（理论最优）。
  • 抖动 (Thrashing)：进程帧数小于工作集，频繁缺页，系统吞吐量暴跌。
缺页中断流程：无效访问 → 中断 → OS调入 → 更新页表 → 恢复执行。
Q2 13:09 知识点
具有TLB的分页系统：逻辑地址→物理地址完整过程
TLB命中 / 未命中 / 缺页
  • TLB命中：直接从TLB取物理帧号，拼接偏移量 → 1次内存访问。
  • TLB未命中：访问内存页表 → 读出帧号 → 更新TLB → 2次内存访问。
  • 缺页：OS从磁盘调入 → 更新页表/TLB → 重新执行指令。
TLB是CPU内部缓存，命中率通常 > 95%。

重组发生在目的主机，不在源主机，也不在中间路由器。

Q5 13:26 知识点
数据链路层"泛洪" vs 网络层？
链路层有泛洪，网络层没有
  • 交换机（L2）：查不到目标MAC → 泛洪（除接收口外所有端口转发）。
  • 路由器（L3）：查不到目标IP → 丢弃 + 返回ICMP不可达，不泛洪。
路由器隔离广播域，防止网络风暴。
Q6 13:31 知识点
中继器与集线器的区别
两者都是物理层设备
  • 中继器：2端口，信号再生放大，延长链路。
  • 集线器：多端口的中继器，收到信号后复制到所有其他端口。
  • 均不识别MAC地址，不分割冲突域。

Q25 14:xx 计算题
[计算] UDP数据报分片：UDP数据字段8192字节，经以太网MTU=1500发送，应当划分为几个数据报片？各片数据字段长度和片偏移字段值为多少？
划分为 6 个数据报片

一、关键参数

参数数值说明
UDP 数据字段长度8192 字节题目给出
UDP 首部长度8 字节固定值
以太网 MTU1500 字节以太网帧数据字段最大长度（含IP首部+IP数据）
IPv4 首部长度20 字节固定值（无选项字段）
每个分片最大数据部分1480 字节1500 - 20 = 1480，且必须是8的倍数（1480 / 8 = 185）

二、需要传输的总数据量

IP层需要传输的数据 = UDP首部 + UDP数据字段 = 8 + 8192 = 8200 字节

IP分片是将整个UDP数据报（含UDP首部）作为数据部分进行分片，总长度是8200字节。

三、分片数量计算

分片数 = ceil(8200 / 1480) = ceil(5.54) = 6 个

前5个分片每个携带1480字节，第6个分片携带剩余部分。

四、各分片数据长度

前5个分片：1480 x 5 = 7400 字节

第6个分片：8200 - 7400 = 800 字节（800 / 8 = 100，合法）

注意：第一个分片包含UDP首部（8字节） + UDP数据前1472字节，因此若问UDP数据字段长度：第1~5片各1472B，第6片800B。

五、片偏移字段计算（以8字节为单位）

分片序号 数据字段长度（字节） 片偏移字段值
1 1472（含UDP首部8B） 0
2 1472 184  (1480 / 8)
3 1472 368  (2960 / 8)
4 1472 552  (4440 / 8)
5 1472 736  (5920 / 8)
6 800 920  (7400 / 8)
片偏移以8字节为单位，表示该分片的数据在原始IP数据报数据部分中的起始偏移位置除以8。

UDP数据报分片计算

以太网MTU分片 · 片偏移计算

Q25 计算题 计算题
一个 UDP 用户数据报的数据字段为 8192 字节，要使用以太网来传送。试问应当划分为几个数据报片？说明每一个数据报片的数据字段长度和片偏移字段的值。
划分为 6 个数据报片

一、关键参数

参数数值说明
UDP 数据字段长度8192 字节题目给出
UDP 首部长度8 字节固定值
以太网 MTU1500 字节含 IP 首部 + IP 数据部分
IPv4 首部长度20 字节固定值（无选项）
每个分片最大数据1480 字节1500 - 20 = 1480，1480 / 8 = 185（合法）

二、IP层总数据量

UDP首部 8B + UDP数据 8192B = IP数据部分 = 8200 B

三、分片数量

ceil(8200 / 1480) = ceil(5.54) = 6 片

前5片各1480B：1480 x 5 = 7400B；第6片：8200 - 7400 = 800B

四、分片结果表

分片 数据字段长度（字节） 片偏移字段值 说明
1 1472（UDP首部8B + 数据1472B） 0 起点为0
2 1472 184 1480 / 8 = 184
3 1472 368 2960 / 8 = 368
4 1472 552 4440 / 8 = 552
5 1472 736 5920 / 8 = 736
6 800 920 7400 / 8 = 920（最后一片）
关键规则：除最后一片外，每个分片的数据长度必须是8字节的倍数；片偏移以8字节为单位。
第一个分片包含完整的UDP首部（8B），后续分片只携带UDP数据的后续部分。重组时只有分片1的UDP首部被保留，其余分片的数据部分直接拼接。

Q15 14:05 判断题
[判断] 家庭上网接入ISP，上传和下载信道不对称
正确 — 家庭宽带通常不对称（ADSL/光纤下载 > 上传）
  • ADSL、光纤宽带：下载速率远高于上传（非对称）。
  • 企业专线：上下行对等（对称）。
Q16 14:08 判断题
[判断] 计算机网络按传输介质带宽分为基带网络和宽带网络
正确 — 基带（数字/单信道）+ 宽带（模拟/多信道FDM）
  • 基带：数字信号，占用全部频带，如以太网。
  • 宽带：模拟信号，频分复用多信道，如ADSL、有线电视。
Q17 14:12 判断题
[判断] IPv6取消了校验和字段，加快路由器处理速度
正确 — IPv6取消首部校验和，减少每跳开销
  • IPv4有首部校验和，每跳需重算（TTL变化）。
  • IPv6取消校验和，减少路由器开销，转发更快。
  • 可靠性由上层协议（TCP/UDP）和链路层保障。
Q18 14:16 判断题
[判断] VLAN是由一些局域网网段构成的与物理位置有关的逻辑组
错误 — VLAN与物理位置无关

VLAN是 与物理位置无关 的逻辑组，可跨交换机、跨楼层划分。

  • 传统局域网：依赖物理位置。
  • VLAN：按部门、项目组等逻辑功能划分。
Q19 14:17 判断题
[判断] 有线电视使用的信道使用了频分复用技术
正确 — 有线电视（CATV）使用FDM

同轴电缆频带（5MHz~1GHz）划分为多个独立频道（各6~8MHz），同时传输多路信号，是 频分复用（FDM） 的典型应用。

Q20 14:18 判断题
[判断] 流量控制和可靠传输也是数据链路层的功能
正确 — 数据链路层具备流量控制和可靠传输
  • 流量控制：滑动窗口协议（停止-等待、后退N帧、选择重传）。
  • 可靠传输：ARQ（自动重传请求）。
  • 数据链路层是 点到点（hop-by-hop），与TCP端到端流量控制不同。
Q21 14:20 判断题
[判断] 网络域名地址便于用户记忆，通俗易懂，可以用中文名称命名
正确 — 域名支持国际化（IDN）
  • 域名比IP地址易记，如 www.baidu.com。
  • 支持中文域名（如"清华大学.中国"），底层转换为Punycode编码。
  • 自2009年ICANN批准IDN以来，中文等非ASCII字符域名已广泛应用。
Q22 14:21 判断题
[判断] 在宽带线路上比特信息传播得快
错误 — 宽带 /= 传播速度快
  • 带宽：单位时间传输的比特数量（传输速率）。
  • 传播速度：信号在介质中行进的物理速度，由介质决定（铜缆/光纤），与带宽无关。
光纤传播速度略慢于铜缆，但带宽远高于铜缆。
Q23 14:22 判断题
[判断] 广播电视使用的信道是单工信道，有线电视使用的信道也是单工信道
正确 — 传统广播电视和有线电视均为单工
  • 单工：数据单向传输（A到B，B不能发）。
  • 广播电视：电视台到用户，单向。
  • 传统有线电视（CATV）：前端到用户，单向。
  • 注意：现代有线电视已升级双向（点播/宽带），但题目考查传统模式，判为单工。
Q24 14:23 判断题
[判断] ARP请求报文是广播报文
正确 — ARP请求为广播，响应为单播
  • ARP请求：目标MAC = FF-FF-FF-FF-FF-FF（广播），询问"谁有该IP？"。
  • ARP响应：单播回复请求者。
  • ARP缓存可减少后续广播次数。
