5原题
一个数据报长度为4000字节（固定首部长度）。现在经过一个网络传送，但此网络能够传送的最大数据长度为1500字节。试问应当划分为几个短些的数据报片？各数据报片的数据字段长度、片偏移字段和MF标志应为何数值？
5解答

【分析】

• IP数据报固定首部长度为 20字节

• 网络MTU = 1500字节，因此每片数据长度 ≤ 1500 - 20 = 1480字节

• 原始数据部分 = 4000 - 20 = 3980字节

• 分片数量 = ⌈3980 ÷ 1480⌉ = ⌈2.69⌉ = 3片


【分片结果】

片1：总长1500B，数据1480B，MF=1，偏移=0

片2：总长1500B，数据1480B，MF=1，偏移=1480÷8=185

片3：总长1040B，数据1020B，MF=0，偏移=2960÷8=370


【验证】

片1数据：字节0~1479

片2数据：字节1480~2959，偏移=185✓

片3数据：字节2960~3979，偏移=370✓

6原题
一个3200bit长的TCP报文传到IP层，加上160bit的首部后成为数据报。下面的互联网由两个局域网通过路由器连接起来。但第二个局域网所能传送的最长数据帧中的数据部分只有1200bit，因此数据报在路由器必须进行分片。试问第二个局域网向其上层要传送多少比特的数据（这里的"数据"当然指局域网看见的数据）？
6解答

【分析】

• TCP报文 = 3200bit，IP首部 = 160bit

• 第二个局域网MTU = 1200bit（含IP首部）

• 每片IP数据部分 ≤ 1200 - 160 = 1040bit

• 片偏移以8字节(64bit)为单位，所以每片数据部分最大取1024bit

• 分片数量 = ⌈3200 ÷ 1024⌉ = ⌈3.125⌉ = 4片


【关键理解】

• "向下传送"：主机→路由器

• "向上传送"：路由器→目标主机

• 每经过一个分片，都需要加上新的IP首部


【计算结果】

第二个局域网向上传送 = TCP数据 + 4 × IP首部

= 3200 + 4 × 160 = 3840bit

解题要点

• 题目中"向下传送"指从主机到路由器，"向上传送"指从路由器到目标主机

• 片偏移必须为 8字节(64bit) 的倍数，因此每片数据最大取 1024 bit

• 分片数量 = ⌈TCP数据 / 每片数据⌉，每个分片都带独立的 IP 首部

• 最终传送总量 = TCP数据 + 分片数 × IP首部
