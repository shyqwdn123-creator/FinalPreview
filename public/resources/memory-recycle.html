情况一 · 题 13-1

释放区与插入点前一分区 F1 相邻接

回收区紧邻前面的空闲分区 F1。此时只需以 F1 的表项作为新表项，修改其大小即可完成回收。

1回收区与 F1 相邻 → 合并到 F1
📐 内存布局示意
📋 空闲分区表
序号始址大小状态

场景：进程释放一块内存区域，该区域紧邻前面的空闲分区 F1。

操作：释放区与 F1 合并为一个更大的空闲分区，起始地址沿用 F1 的始址，大小 = 原 F1 大小 + 释放区大小。

新分区始址 = F1.始址（不变）
新分区大小 = F1.大小 + 释放区.大小

分区表操作：以 F1 的表项为新表项，但修改新表项的大小。不增加表项数量。

✅ 答案：C — 以F1分区的表项为新表项，但修改新表项的大小

情况二 · 题 13-2

释放区与插入点后一分区 F2 相邻

回收区紧邻后面的空闲分区 F2。此时需要以 F2 的表项作为新表项，同时修改其大小和始址。

2回收区与 F2 相邻 → 合并到 F2
📐 内存布局示意
📋 空闲分区表
序号始址大小状态

场景：进程释放一块内存区域，该区域紧邻后面的空闲分区 F2。

操作：释放区与 F2 合并为一个更大的空闲分区，起始地址改为释放区的始址，大小 = 释放区大小 + 原 F2 大小。

新分区始址 = 释放区.始址（注意：变了！）
新分区大小 = 释放区.大小 + F2.大小

分区表操作：以 F2 的表项作为新表项，同时修改新表项的大小和始址。不增加表项数量。

✅ 答案：D — 以F2分区的表项作为新表项，同时修改新表项的大小和始址

情况三 · 题 13-3

释放区不与 F1 和 F2 相邻接

回收区前后都是已分配分区。此时需要为回收区建立一个新的分区表项，填入其大小和始址。

3回收区与 F1、F2 均不相邻 → 新建表项
📐 内存布局示意
📋 空闲分区表
序号始址大小状态

场景：进程释放一块内存区域，该区域前后都是已分配分区（与任何空闲分区都不相邻）。

操作：释放区自己成为一个新的空闲分区，需要在空闲分区表中新增一条记录。

新建表项 → 始址 = 释放区.始址
新建表项 → 大小 = 释放区.大小

分区表操作：为回收区建立分区表项，填上分区的大小和始址。空闲分区表项数 +1。

✅ 答案：A — 为回收区建立分区表项，填上分区的大小和始址

情况四 · 题 13-4

释放区既与 F1 相邻接，又与 F2 相邻接

回收区恰好夹在两个空闲分区之间。此时需要以 F1 的表项为新表项，修改其大小，并删除 F2 的表项，将三块合并为一。

4回收区与 F1、F2 均相邻 → 三区合并
📐 内存布局示意
📋 空闲分区表
序号始址大小状态

场景：进程释放一块内存区域，该区域恰好夹在两个空闲分区之间——前面是 F1，后面是 F2。

操作：F1 + 释放区 + F2 三块合并为一个更大的空闲分区。始址沿用 F1 的始址，大小 = F1 + 释放区 + F2。

新分区始址 = F1.始址（不变）
新分区大小 = F1.大小 + 释放区.大小 + F2.大小
⚠️ 同时删除 F2 对应的表项！

分区表操作：以 F1 的表项为新表项，修改其大小，还要删除 F2 所对应的表项。表项数 -1。

✅ 答案：E — 以F1分区的表项为新表项，但修改新表项的大小且还要删除F2所对应的表项

📊 四种情况总结对比

情况一：与 F1 相邻

释放区紧邻前面的空闲分区 F1

修改 F1 表项 → 大小 += 释放区大小

表项数：不变 | 始址：不变（F1 的始址）

情况二：与 F2 相邻

释放区紧邻后面的空闲分区 F2

修改 F2 表项 → 始址 = 释放区始址，大小 += 释放区大小

表项数：不变 | 始址：变为释放区的始址

情况三：均不相邻

释放区前后都是已分配分区

新建 表项 → 填入释放区的始址和大小

表项数：+1 | 独立成为一个新的空闲分区

情况四：与 F1、F2 均相邻

释放区恰好夹在两个空闲分区之间

修改 F1 表项大小 + 删除 F2 表项

表项数：-1 | 三块合并为一个大的空闲分区

记忆口诀：前合改大小 · 后合改始址 · 不合新建表 · 都合删后项
